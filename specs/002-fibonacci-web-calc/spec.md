# 기능 사양서: 피보나치 웹 계산기

**피처 브랜치**: `002-fibonacci-web-calc`  
**생성일**: 2026-02-10  
**상태**: 초안  
**입력**: 사용자 설명: "사용자의 입력을 받아 fibonazzi 수열을 계산해서 사용자에게 보여주는 SW를 만들어야 해. 사용자는 web browser를 통해 입력을 줄 수 있고, 계산된 결과를 web browser를 통해 제공해야 해. 모든 문서는 한글로 만들어 주고, constitution.md 파일에 해당 조항을 업데이트 해줘"

## 사용자 시나리오 및 테스트 *(필수)*

### 사용자 스토리 1 - N번째 피보나치 수 계산 (우선순위: P1)

사용자로서, 웹 브라우저에 음이 아닌 정수를 입력하고 그에 해당하는 피보나치 수를 즉시 확인하여 수학적 결과를 얻고 싶습니다.

**우선순위 이유**: 소프트웨어의 핵심 가치를 전달하는 가장 기본적인 기능입니다.

**독립적 테스트**: 입력창에 "5"를 입력하고 결과값으로 "5"(0, 1, 1, 2, 3, 5 수열의 5번째 값)가 표시되는지 확인하여 테스트할 수 있습니다.

**인수 시나리오**:
1. **Given** 웹 애플리케이션이 로드된 상태에서, **When** 사용자가 "0"을 입력하면, **Then** 결과값 "0"이 표시됩니다.
2. **Given** 웹 애플리케이션이 로드된 상태에서, **When** 사용자가 "10"을 입력하면, **Then** 결과값 "55"가 표시됩니다.

---

### 사용자 스토리 2 - 입력 유효성 검사 및 오류 피드백 (우선순위: P2)

사용자로서, 음수나 텍스트와 같은 잘못된 값을 입력했을 때 명확한 오류 메시지를 받아 왜 계산을 수행할 수 없는지 이해하고 싶습니다.

**우선순위 이유**: 애플리케이션의 견고함을 보장하고 잘못된 입력으로 인한 시스템 장애를 방지하여 사용자 경험을 향상시킵니다.

**독립적 테스트**: "-1" 또는 "abc"를 입력하고 적절한 오류 메시지가 표시되는지 확인하여 테스트할 수 있습니다.

**인수 시나리오**:
1. **Given** 웹 애플리케이션이 로드된 상태에서, **When** 사용자가 음수를 입력하면, **Then** "음이 아닌 정수를 입력해주세요"라는 오류 메시지가 표시됩니다.
2. **Given** 웹 애플리케이션이 로드된 상태에서, **When** 사용자가 숫자가 아닌 문자를 입력하면, **Then** "유효하지 않은 입력입니다. 숫자를 입력해주세요"라는 오류 메시지가 표시됩니다.

---

### 예외 케이스(Edge Cases)

- **매우 큰 입력값**: 사용자가 100을 초과하는 숫자를 입력할 경우 "최대 100까지만 입력 가능합니다"라는 안내와 함께 계산을 제한합니다.
- **빈 입력값**: 값을 입력하지 않고 계산 버튼을 눌렀을 때 어떻게 처리합니까? (가정: 무시하거나 입력을 요구하는 메시지 표시)
- **소수점 입력**: "5.5"와 같은 소수를 입력할 경우 어떻게 처리합니까? (가정: 정수로 내림 처리하거나 유효하지 않은 입력으로 간주)

## 요구사항 *(필수)*

### 기능적 요구사항

- **FR-001**: 시스템은 숫자 입력 필드와 "계산" 버튼이 포함된 웹 기반 사용자 인터페이스를 제공해야 합니다.
- **FR-002**: 시스템은 입력값 $n$에 대해 $F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)$ 공식을 사용하여 $n$번째 피보나치 수를 계산해야 합니다.
- **FR-003**: 시스템은 계산 결과를 동일한 페이지에 명확하게 표시해야 합니다.
- **FR-004**: 시스템은 계산을 수행하기 전에 입력값이 음이 아닌 정수인지 검증해야 합니다.
- **FR-005**: 시스템은 결과를 표시할 때 $n$번째 피보나치 숫자와 그 앞의 5개 숫자를 함께 표시하여 계산의 맥락을 제공해야 합니다. (단, $n < 5$인 경우 가능한 범위까지만 표시)

### 주요 엔티티

- **계산 요청(Calculation Request)**: 입력값과 요청 시간을 포함하는 단일 피보나치 계산 시도를 나타냅니다.
- **피보나치 결과(Fibonacci Result)**: 입력에 대응하는 출력값 또는 오류 발생 시 오류 메시지를 나타냅니다.

## 성공 기준 *(필수)*

### 측정 가능한 결과

- **SC-001**: 사용자는 $n \le 40$ 인 입력에 대해 500ms 이내에 계산 결과를 받아야 합니다.
- **SC-002**: 잘못된 입력(음수, 텍스트, 소수 등)에 대해 시스템 다운 없이 100% 오류 메시지를 제공해야 합니다.
- **SC-003**: 최신 버전의 Chrome, Firefox, Edge 브라우저에서 올바르게 렌더링되고 작동해야 합니다.
- **SC-004**: 사용자는 입력부터 결과 확인까지 3번 이하의 클릭/상호작용으로 완료할 수 있어야 합니다.

## 가정 사항
- 피보나치 수열은 $F(0)=0, F(1)=1$로 시작한다고 가정합니다.
- 별도의 사용자 인증(로그인)은 필요하지 않은 공개 도구로 가정합니다.
